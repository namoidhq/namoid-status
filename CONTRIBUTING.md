# Contributing to NamoID

Thanks for your interest in improving NamoID! This repo is the
[Upptime](https://upptime.js.org)-powered status page for NamoID, and is one of
several open-source projects under the
[`namoidhq`](https://github.com/namoidhq) org.

## Filing issues

Found a bug or have a feature request? Open an issue in this repo's
[Issues](../../issues) tab. Note that the uptime incidents Upptime opens here
are generated automatically — those aren't human-filed. Security issues should
**not** be filed publicly — see [SECURITY.md](./SECURITY.md).

## Submitting a pull request

1. Fork and create a branch from `main`.
2. Make your change. Keep it focused — one logical change per PR.
3. Open the PR and link the related issue (e.g. `#123`).

## Local development

Most configuration lives in [`.upptimerc.yml`](./.upptimerc.yml) — the list of
monitored endpoints, the site, and notification settings. The GitHub Actions
workflows in `.github/workflows/` run the checks and rebuild the site on a
schedule; there's little to run locally. See the
[Upptime docs](https://upptime.js.org/docs/) for configuration details.
